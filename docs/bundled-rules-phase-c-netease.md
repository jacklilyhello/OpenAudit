Phase C commits real pinned NetEase G79 and X19 snapshots, deterministic generated Packs, reports, provenance, and maintainer synchronization tooling. It remains default-disabled and local-only: OpenAudit service startup never downloads or updates bundled rules.

## Offline reproducibility

Normal CI does not need network access:

```sh
make verify-bundled-netease
```

This command reads committed snapshots, regenerates G79/X19 Packs and reports in memory, validates `SOURCE.json` with typed JSON decoding, compares generated bytes and hashes with committed artifacts, and fails if anything is stale.

To refresh generated artifacts from committed snapshots:

```sh
make regenerate-bundled-netease
git diff --exit-code
```

## Maintainer synchronization

The dedicated maintainer command is:

```sh
go run ./cmd/sync-netease-rules -mode=verify
go run ./cmd/sync-netease-rules -mode=regenerate
go run ./cmd/sync-netease-rules -mode=download \
  -commit 97d3236b999c5f95c7adac1debf7fffb81d8bda2 \
  -expect-g79-sha256 f398f68c5af1b2b92e01b8dc895ddecef86b72cf2783c798566bab604b5aa07d \
  -expect-x19-sha256 5f8f7e4d6b2436a4b47e5d8c5fa2e936d06b6c18e45689ebf8aa712097baeba4 \
  -expect-license-sha256 3972dc9744f6499f0f9b2dbf76696f2ae7ad8af9b23dde66d6af86c9dfb36986
```

Download mode rejects floating refs, only accepts the reviewed full 40-character pinned commit, downloads only the three allowlisted upstream paths over HTTPS, enforces finite timeouts, response status checks, response-size limits, and a restrictive redirect policy, validates JSON/LICENSE content before replacement, stages through temporary files, and preserves prior snapshots on failure. It does not forward credentials and is never invoked by the OpenAudit server.
